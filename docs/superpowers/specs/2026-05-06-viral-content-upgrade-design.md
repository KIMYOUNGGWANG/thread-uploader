# CosmicPath 바이럴 콘텐츠 업그레이드 설계

**작성일:** 2026-05-06  
**상태:** 승인됨  
**목표:** Threads 텍스트 전용 콘텐츠의 바이럴 도달률 향상

---

## 배경 및 문제 정의

현재 상황:
- 팔로워 520명, 포스트당 조회수 100~1000회
- 조회수 범위가 크다는 건 일부 포스트가 팔로워 외부까지 도달 중 — 그러나 공유/저장/팔로우로 이어지지 않음
- 이미지 전략 없음 (의도적), 텍스트 전용

진단된 병목 3가지:
1. **모델(Haiku)이 공식 instruction을 자주 무시** — "3번은 반드시 반전" 같은 뉘앙스를 약 50% 확률로 놓침
2. **태그유도 공식 부재** — 현재 10개 공식 중 "친구한테 보내고 싶다" 심리를 유발하는 게 없음
3. **정적 토픽** — 지금 시점의 트렌딩 이슈와 연결되지 않아 콘텐츠가 "언젠가 쓴 글"처럼 느껴짐

---

## 변경 범위

### 1. 생성 모델 업그레이드

**파일:** `src/app/api/generate/route.ts`

- `claude-haiku-4-5-20251001` → `claude-sonnet-4-6`
- Sonnet은 복잡한 formula instruction(뉘앙스, 반전, 구조적 제약)을 훨씬 더 정확하게 실행
- 비용: 포스트 30개 생성 기준 3~5배 증가하지만 절대 금액은 미미한 수준
- max_tokens: 900 유지 (콘텐츠 길이 기준 충분)

### 2. 바이럴 공식 3개 추가

DB의 `Brand.brandConfig.formulas` 배열에 추가. 기존 공식들과 함께 가중치 풀에 포함됨.

#### 2-1. `tag_friend` — 태그유도형 (weight: 3)

```
특정 사주/별자리 유형의 사람을 너무 정확하게 묘사해서
"이거 완전 내 친구 얘기다" 반응을 유발하는 포스트.

구조:
- 오프닝: "[특정 유형] 사람들 있잖아. 이거 들으면 알 거야."
- 본문: 그 유형 특유의 행동 패턴 3-4가지를 아주 구체적으로 묘사
  (추상적 설명 금지 — "이럴 때 이렇게 한다" 장면 묘사 필수)
- 마무리: "주변에 이런 사람 있으면 몰래 보내줘"
  또는 "이거 해당되는 사람 태그하고 싶어졌지?"

핵심: 묘사가 충격적으로 정확해야 함. 독자가 자기 자신 또는
아는 사람을 즉시 떠올릴 수 있을 것.
```

#### 2-2. `prophecy` — 예언형 (weight: 3)

```
"이번 [달/주/시기]에 [특정 사주/별자리 유형]에게 반드시 일어날 일"
구조로 시간 제한 긴박감을 만드는 포스트.

구조:
- 오프닝: "이번 달 [타입]들한테 무조건 생기는 일 알아?" 또는
  "지금 이 글 보이면 이유 있는 거야"
- 본문: 구체적 상황 2-3가지 예언 (막연한 "좋은 일이 생긴다" 금지
  — "갑자기 연락이 오거나", "결정을 강요받는 상황" 등 구체적으로)
- 마무리: "저장해두고 월말에 확인해봐"

핵심: 저장 CTA가 자연스러워야 함. 시간이 지나면 확인할 수 있다는
구조가 저장 동기를 만듦.
```

#### 2-3. `insider` — 내부자폭로형 (weight: 3)

```
"사주를 많이 본 사람만 아는 것", "점집에서 말 안 해주는 이유"
구조로 독자를 희귀 정보의 수혜자로 만드는 포스트.

구조:
- 오프닝: "사주 오래 본 사람들은 다 아는데" 또는
  "이거 아무도 대놓고 말 안 해줌"
- 본문: 일반적으로 알려지지 않은 사주/점성술 인사이트 2-3가지
  각 항목은 "표면적 설명 → 진짜 이유"로 구성
- 마무리: "알면 알수록 신기한 게 사주야" 또는
  "이거 스크린샷 해두면 나중에 써먹음"

핵심: "내가 특별한 정보를 먼저 알게 됐다"는 느낌을 줘야 함.
스크린샷/저장/공유 동기는 여기서 발생.
```

### 3. 트렌딩 토픽 주입 시스템

#### 3-1. 타입 변경

**파일:** `src/types/brand.ts`

`BrandConfig` 인터페이스에 옵셔널 필드 추가:

```typescript
interface BrandConfig {
  // 기존 필드들 유지
  trendingTopics?: string[];  // 매주 업데이트하는 시즌/이슈 키워드
}
```

#### 3-2. Generate API 변경

**파일:** `src/app/api/generate/route.ts`

토픽 풀 합산:
```typescript
const allTopics = [
  ...config.topics,
  ...(config.trendingTopics ?? []),
];
const topics = shuffleTopics(allTopics, count);
```

#### 3-3. 브랜드 설정 UI 변경

**파일:** `src/components/BrandSettingsForm.tsx` + `src/app/brands/[slug]/settings/page.tsx`

"이번 주 트렌딩 토픽" 섹션 추가:
- textarea 입력 (줄바꿈으로 구분)
- 저장 시 `brandConfig.trendingTopics` 업데이트
- 예시 표시: "수성 역행 2026년 5월", "쌍둥이자리 시즌 시작", "2026 삼재해 해당 띠"

운영 방식: 매주 1회 수동 업데이트. 자동화 불필요.
