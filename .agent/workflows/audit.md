# 🚀 /audit - 출시 준비 점검 (Launch Readiness Audit)
## 🎯 목적
프로젝트의 **수익화 및 서비스 안정성**을 위한 최종 점검을 수행합니다.
## 📋 10대 핵심 점검 영역
### 1. 인증 & 사용자 ID
- 데이터베이스 영속성, 온보딩 흐름, 유료 경로 보호, 소셜 로그인
### 2. 데이터베이스 & 정합성
- 스키마 연결, RLS 정책, 백업 설정
### 3. 결제 & 크레딧
- 웹훅 서명 검증, 즉시 크레딧 지급, 잔액 부족 처리, 플랜별 제한
### 4. 이메일 & 트랜잭션
- 발송 API 설정, 이벤트별 발송, 도메인 인증
### 5. AI & LLM 통합
- API Key 보안, 타임아웃 처리, 에러 시 환불, 프롬프트 보안, 비용 제어
### 6. 환경 & 보안
- 환경 변수 관리, 민감 로그 제거, Rate Limiting
### 7. SEO & OG 태그
- 메타 태그, 소셜 이미지, sitemap.xml
### 8. 법적 고지
- 이용약관, 개인정보처리방침, 환불 정책
### 9. 에러 핸들링
- 404/500 커스텀 페이지, Sentry 초기화
### 10. 배포 체크
- 빌드 성공, 모바일 반응형, 애셋 최적화
## 📦 산출물
- **Launch Readiness Score**: 전체 준비도 점수 (예: 85%)
- **TOP 3 Critical Fixes**: 오늘 반드시 해결해야 할 문제
- **보일러플레이트 목록**: 아직 커스터마이징 안 된 파일들
