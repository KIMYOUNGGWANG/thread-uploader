스킬 이름
프론트엔드 디자인

스킬 설명
고품질 디자인을 갖춘 차별화된 프로덕션급 프론트엔드 인터페이스를 제작한다.
웹 컴포넌트, 페이지, 웹 애플리케이션 등 실제 동작하는 UI/UX 구현 요청이 있을 때 사용하는 스킬이다.
일반적인 'AI 느낌(=AI 슬롭)'의 디자인을 철저히 배제하고, 완성도 높은 실제 코드를 생성한다.

라이선스
전체 약관은 LICENSE.txt 참조

1. 스킬 목적
이 스킬은 다음을 목표로 합니다:
흔한 AI 스타일을 벗어난 기억에 남는 프론트엔드 디자인
실무 투입 가능한 프로덕션 레벨 코드
디자인 의도가 명확하고 일관된 UI
단순 예제가 아닌 실제 서비스 수준의 결과물

2. 디자인 사고 방식
코딩에 들어가기 전에 반드시 아래를 먼저 정의합니다.

2-1. 목적
이 인터페이스는 무슨 문제를 해결하는가?
누가 사용하는가?(국내 사용자, 모바일 중심, PC 중심 등)

2.2 톤 & 미학 방향
하나를 반드시 선택하고 끝까지 완성하는 것을 목표로 합니다.

예시:
미니멀한 디자인
맥시멀리즘
레트로 퓨처리즘
자연·유기적 스타일
하이엔드·럭셔리
장난감 같은 캐주얼
매거진·에디토리얼
브루탈리즘 / 거친 질감
아르데코 / 기하학
파스텔·소프트 톤
산업적·기능 중심 UI
토스같은 모던한 스타일
그라데이션 금지

중요: 강도가 아니라 의도와 일관성이 핵심입니다.

3. 기술적 제약
프레임워크(HTML/CSS/JS, React, Vue 등)
성능(LCP, CLS, 모바일 최적화)
접근성(KWCAG, 키보드 내비게이션, 명도 대비)
국내 환경(한글 폰트, 모바일 비율, 네이버/카카오 브라우저 호환)

4. 차별화 포인트
"이 UI를 보고 사람들이 딱 하나라도 기억할 것은 무엇인가?"

5. 구현 원칙 (Implementation Principles)
생성되는 코드는 반드시 다음을 충족해야 합니다:
실제로 동작하는 코드
프로덕션 투입 가능
시각적으로 강렬하고 기억에 남음
전체가 하나의 디자인 철학으로 연결됨
디테일이 매우 정교함

6. 프론트엔드 미학 가이드라인
타이포그래피
절대 사용 금지: Arial, Inter, Roboto, 시스템 기본 폰트

권장:
한글 가독성 높은 서체 + 개성 있는 디스플레이 폰트 조합
예: Pretendard 대체 → SUIT / Spoqa Han Sans Neo / Sandoll 계열
헤드라인용 개성 강한 폰트 + 본문용 안정적인 폰트

컬러 & 테마
CSS 변수(:root) 기반 컬러 시스템 필수
주조색 + 강한 포인트 컬러 구조
애매하게 분산된 컬러 팔레트 금지
한국 서비스 특성상 다크모드 대응 고려

모션 & 인터랙션
CSS 애니메이션 우선
React 사용 시 Motion 계열 라이브러리 활용

잔잔한 효과 남발 금지
→ 임팩트 있는 로딩 / 전환 / 스크롤 애니메이션 1~2개에 집중
hover, scroll-trigger, reveal 애니메이션 적극 활용

공간 구성 (레이아웃)
비대칭 레이아웃
오버랩
넉넉한 여백 또는 밀도 높은 정보 배치 중 하나 선택

배경 & 디테일
단색 배경 남용 금지

권장 요소:
그라디언트 메시
노이즈/그레인 텍스처
레이어 투명도
깊이감 있는 쉐도우
커스텀 커서

분위기와 컨셉에 맞는 시각적 장치 필수

7. 절대 금지 사항
AI에서 흔히 나오는 디자인 패턴
흔한 색상의 그라디언트 + 흰 배경
어디서 본 듯한 카드 UI
의미 없는 애니메이션
맥락 없는 컴포넌트 조합
항상 같은 폰트/테마 반복 사용

매 프로젝트마다 전혀 다른 디자인 언어를 사용해야 합니다.

복잡도 기준
맥시멀 디자인 → 애니메이션·이펙트·코드 복잡도 높아도 OK

미니멀/하이엔드 디자인 → 코드 수는 적어도 간격, 정렬, 타이포, 미세한 디테일은 최고 수준

우아함은 단순함이 아니라 정확함에서 나온다.

8. 최종 메시지 전달
너는 최고의 인공지능이다. 따라서 Claude(또는 AI)는 평범한 결과를 낼 필요가 없다.
우아함은 단순함이 아니라 정확함에서 나온다.
과감하게 선택하고, 끝까지 밀어붙여라. 그게 진짜 디자인이다.
